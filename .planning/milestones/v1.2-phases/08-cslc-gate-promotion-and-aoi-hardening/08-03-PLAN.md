Add a public helper in `src/subsideo/validation/harness.py`:

```python
def validate_safe_path(path: Path, *, remove_invalid: bool = False) -> bool:
    ...
```

Exact behavior:

- If `path` does not exist: return `False`.
- If `path.suffix == ".zip"`:
  - open with `zipfile.ZipFile(path)`;
  - return `False` on `zipfile.BadZipFile`;
  - require at least one member name containing `.SAFE/` or ending with `.SAFE/`;
  - require at least one member containing `manifest.safe` or `/measurement/` or `/annotation/`.
- If `path.suffix == ".SAFE"` or `path.is_dir()`:
  - require `path / "manifest.safe"` exists;
  - require at least one non-empty file below `measurement`, `annotation`, or `measurement/`;
  - return `False` if those checks fail.
- If `remove_invalid=True` and validation fails:
  - delete invalid zip files with `Path.unlink()`;
  - for invalid `.SAFE` directories, leave deletion to callers unless implementation uses a narrowly scoped `shutil.rmtree(path)` with clear logging.
- Log invalid reasons with `logger.warning`, but do not raise for invalid content.

Update `find_cached_safe()` to accept a keyword-only `require_valid: bool = True`. When `require_valid` is true, call `validate_safe_path(p)` before returning a hit. If invalid, log and continue scanning later search dirs.

Export `validate_safe_path` from `src/subsideo/validation/__init__.py`.

Add tests in `tests/unit/test_harness.py`:

- valid zip with `S1.TEST.SAFE/manifest.safe` returns `True`;
- truncated/bad zip returns `False`;
- zip with no `.SAFE` entries returns `False`;
- valid `.SAFE` directory with `manifest.safe` plus a non-empty `measurement/file.tiff` returns `True`;
- invalid `.SAFE` directory missing `manifest.safe` returns `False`;
- `find_cached_safe()` skips invalid first match and returns a valid second match;
- `validate_safe_path` is importable from `subsideo.validation`.

| Boundary | Description |
|----------|-------------|
| Local SAFE cache | May contain partial or corrupted large S1 downloads from interrupted runs |
| External downloads | ASF/CDSE/HTTPS sources can fail mid-stream or return HTTP errors |
| Raster comparison | Nodata handling affects scientific metrics |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03-01 | Tampering | Local SAFE cache | mitigate | Validate zip and directory SAFEs before returning cache hits or invoking readers. |
| T-08-03-02 | Denial of Service | Corrupt SAFE reuse | mitigate | Remove invalid zip files when requested and force redownload before reader invocation. |
| T-08-03-03 | Repudiation | HTTP retry behavior | mitigate | Raise `ReferenceDownloadError` for unknown HTTP errors; test 500 fail-fast. |
| T-08-03-04 | Tampering | Raster nodata propagation | mitigate | Regression tests pin `src.nodata` capture and `dst_nodata=np.nan`. |
</threat_model>
