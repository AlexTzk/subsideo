         ```python
         runconfig = {
             "runconfig": {
                 "name": "cslc_s1_workflow",

                         "burst_id": cfg.burst_id,

                     "dynamic_ancillary_file_group": {

                     },

                         "product_path": str(cfg.output_dir),
                         "scratch_path": str(cfg.output_dir / "scratch"),
                         "product_version": cfg.product_version,
                     },
                     "primary_executable": {
                         "product_type": "CSLC_S1",
                     },
                 },
             }
         }
         ```
       - Write using `ruamel.yaml` YAML() round-trip mode
       - Return the output_yaml path

    2. `validate_cslc_product(hdf5_paths: list[Path]) -> list[str]`:
       - Per D-12: lightweight validation that fails loudly.
       - For each HDF5 path, check:
         a. File exists and is readable
         b. Opens with h5py.File without error
         c. Contains a `/data` group (top-level data container)
         d. Contains at least one dataset under `/data` (e.g., VV or HH polarization)
         e. Contains `/metadata` group or `/identification` group (OPERA spec metadata)
       - Use h5py (lazy import) for all HDF5 inspection
       - Return list of error strings (empty = valid)
       - Note: exact HDF5 paths may vary by compass version (per RESEARCH.md Open Question 2). Check for `/data` group existence rather than hardcoded dataset paths.

    3. `run_cslc(safe_paths: list[Path], orbit_path: Path, dem_path: Path, burst_ids: list[str] | None, output_dir: Path, tec_file: Path | None = None, product_version: str = "0.1.0") -> CSLCResult`:
       - Per D-04/D-05: config-driven compass execution via Python API
       - Steps:
         a. Create CSLCConfig from args
         b. `output_dir.mkdir(parents=True, exist_ok=True)`
         c. `(output_dir / "scratch").mkdir(parents=True, exist_ok=True)`
         d. Call `generate_cslc_runconfig(cfg, output_dir / "cslc_runconfig.yaml")`
         e. Lazy import and call compass:
            ```python
            from compass.s1_cslc import run as compass_run
            compass_run(run_config_path=str(runconfig_yaml), grid_type="geo")
            ```
         f. Glob `output_dir / "*.h5"` for output HDF5 files
         g. Validate: `errors = validate_cslc_product(h5_paths)`
         h. Return CSLCResult with fields populated
       - Wrap compass calls in try/except, logging errors via loguru
       - Import CSLCConfig and CSLCResult from subsideo.products.types
       - Derive burst_ids for result: use cfg.burst_id if provided, else extract from output filenames

    All imports of compass and h5py must be lazy (inside functions) since compass is conda-forge-only.
