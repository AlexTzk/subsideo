         ```python
         runconfig = {
             "runconfig": {
                 "name": "rtc_s1_workflow",

                     "input_file_group": {
                         "safe_file_path": [str(p) for p in cfg.safe_file_paths],
         ```

       - Write using `ruamel.yaml` YAML() round-trip mode (same pattern as config.py dump_config)
       - Return the output_yaml path

    2. `ensure_cog(input_tif: Path, output_cog: Path | None = None) -> Path`:
       - Per D-10 and Pitfall 1: opera-rtc GeoTIFF is NOT COG. Must post-process.
       - Use `rio_cogeo.cogeo.cog_translate` with DEFLATE profile:
         ```python
         from rio_cogeo.cogeo import cog_translate
         from rio_cogeo.profiles import cog_profiles
         output_cog = output_cog or input_tif.with_suffix(".cog.tif")
         profile = cog_profiles.get("deflate")
         cog_translate(str(input_tif), str(output_cog), profile,
                       overview_level=5, overview_resampling="nearest",
                       use_cog_driver=True)
         ```
       - Return output_cog path

    3. `validate_rtc_product(cog_paths: list[Path]) -> list[str]`:
       - Per D-12: lightweight validation that fails loudly.
       - For each COG path, check:
         a. File exists and is readable
         b. `rio_cogeo.cog_validate.cog_validate(str(p))` returns is_valid=True
         c. CRS is a UTM projection (EPSG between 32601-32660 or 32701-32760)
         d. Pixel size is approximately 30m (between 25 and 35)
       - Use rasterio to read CRS and transform
       - Return list of error strings (empty = valid)

    4. `run_rtc(safe_paths: list[Path], orbit_path: Path, dem_path: Path, burst_ids: list[str], output_dir: Path, product_version: str = "0.1.0") -> RTCResult`:
       - Per D-01/D-02: thin wrapper around opera-rtc Python API
       - Steps:
         a. Create RTCConfig from args
         b. `output_dir.mkdir(parents=True, exist_ok=True)`
         c. Call `generate_rtc_runconfig(cfg, output_dir / "rtc_runconfig.yaml")`
         d. Lazy import and call opera-rtc:
            ```python
            from rtc.runconfig import RunConfig
            from rtc.rtc_s1 import run_parallel
            opera_cfg = RunConfig.load_from_yaml(str(runconfig_yaml))
            run_parallel(opera_cfg, str(output_dir / "rtc.log"), True)
            ```
         e. Glob `output_dir / "*.tif"` for output files
         f. COG-convert each: `cog_paths = [ensure_cog(p) for p in output_tifs]`
         g. Validate: `errors = validate_rtc_product(cog_paths)`
         h. Return RTCResult(output_paths=cog_paths, runconfig_path=runconfig_yaml, burst_ids=burst_ids, valid=len(errors)==0, validation_errors=errors)
       - Wrap opera-rtc calls in try/except, logging errors via loguru
       - Import RTCConfig and RTCResult from subsideo.products.types

    All imports of opera-rtc and rio-cogeo must be lazy (inside functions) since they are conda-forge-only dependencies.
