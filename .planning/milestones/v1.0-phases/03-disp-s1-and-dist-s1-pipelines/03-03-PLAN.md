```python
"""DISP-S1 product validation against EGMS EU Ortho reference.

Downloads EGMS Ortho products via EGMStoolkit, then compares vertical velocity
component: projects subsideo LOS velocity to vertical using incidence angle,
then compares against EGMS Ortho vertical displacement field.
Grid alignment reprojects subsideo output to match the EGMS reference grid.
"""
from __future__ import annotations

from pathlib import Path

import numpy as np
import rasterio
from loguru import logger
from rasterio.warp import Resampling, reproject

from subsideo.products.types import DISPValidationResult
from subsideo.validation.metrics import bias, spatial_correlation
```

Functions to implement:

- `fetch_egms_ortho(bbox: tuple[float, float, float, float], output_dir: Path) -> Path` -- per D-09, download EGMS Ortho product for a bounding box using EGMStoolkit. Parameters: `bbox` is (west, south, east, north) in WGS84 degrees. Steps:
  1. `output_dir.mkdir(parents=True, exist_ok=True)`.
  2. Lazy import: `import EGMStoolkit` (or the appropriate submodule -- read the installed package to determine the correct import). On ImportError, raise `ImportError("EGMStoolkit is not installed. Install via pip: pip install EGMStoolkit")`.
  3. Use EGMStoolkit to download Ortho-level EGMS products for the given bounding box to `output_dir`. The toolkit's download function typically accepts bbox coordinates and a product level parameter. Use product level "Ortho" (vertical displacement).
  4. Find the downloaded GeoTIFF(s) in output_dir: `sorted(output_dir.glob("*.tif"))`. If multiple tiles, use the first one (or merge if the toolkit produces multiple files -- check what it returns).
  5. Return the path to the downloaded EGMS Ortho GeoTIFF.
  6. Log: `"Downloaded EGMS Ortho product for bbox {bbox} to {output_dir}"`.
  If the EGMStoolkit API differs from the assumed pattern, adapt accordingly. The key requirement is: given a bounding box, download the EGMS Ortho vertical displacement product and return its file path.

- `_los_to_vertical(los_velocity: np.ndarray, incidence_angle: np.ndarray | float) -> np.ndarray` -- per D-08, compute `v_vert = v_los / cos(theta)` where theta is in radians. If `incidence_angle` is a float, convert from degrees to radians. If it's an array, assume already in degrees and convert element-wise. Handle division by zero by returning NaN where `cos(theta) == 0`. Return the vertical velocity array.

- `compare_disp(product_path: Path, egms_ortho_path: Path, incidence_angle_path: Path | None = None, mean_incidence_deg: float = 33.0) -> DISPValidationResult` -- per D-10, the main comparison function:
  1. **Load EGMS reference grid** (target grid per Pitfall 4): `rasterio.open(egms_ortho_path)`, read band 1 as float64, store transform and CRS.
  2. **Reproject subsideo product to EGMS grid** (per compare_rtc.py pattern): create `prod_aligned = np.empty_like(egms_data)`, call `reproject(source=rasterio.band(prod, 1), destination=prod_aligned, dst_transform=egms_transform, dst_crs=egms_crs, resampling=Resampling.bilinear)`.
  3. **Load or construct incidence angle**: If `incidence_angle_path` is provided, open it with rasterio and reproject to EGMS grid (same as product). Otherwise use `mean_incidence_deg` as a scalar float. Per Pitfall 3: use per-pixel when available, fall back to mean.
  4. **Project LOS to vertical**: call `_los_to_vertical(prod_aligned, incidence_angle)`.
  5. **Mask to intersection**: create a valid mask where both `vert_velocity` and `egms_data` are finite (not NaN). Per CONTEXT.md specifics: handle partial EGMS overlap by computing metrics only over the intersection. Set non-overlapping pixels to NaN in both arrays.
  6. **Compute metrics** using shared functions: `corr = spatial_correlation(vert_velocity, egms_data)`, `bias_val = bias(vert_velocity, egms_data)`.
  7. Log: `"DISP validation: r={corr:.4f}, bias={bias_val:.2f} mm/yr"`.
  8. Return `DISPValidationResult(correlation=corr, bias_mm_yr=bias_val, pass_criteria={"correlation_gt_0.92": corr > 0.92, "bias_lt_3mm_yr": abs(bias_val) < 3.0})`.
