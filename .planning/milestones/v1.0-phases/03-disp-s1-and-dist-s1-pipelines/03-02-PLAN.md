<objective>
Implement the DIST-S1 surface disturbance pipeline orchestrator with both low-level and AOI entry points.

Purpose: PROD-05 requires producing DIST-S1 surface disturbance products from RTC time series. This plan creates `products/dist.py` as a thin wrapper around `dist_s1.run_dist_s1_workflow()`, with conditional lazy import for the conda-forge-only dist-s1 package. Per D-12, also creates `run_dist_from_aoi()` that takes AOI+date range, resolves MGRS tiles, runs RTC first, then feeds into dist-s1. DIST output validation is structural (COG compliance) only -- no EU-specific reference dataset exists for DIST.

Output: `products/dist.py` with `run_dist()` and `run_dist_from_aoi()`, DISTConfig/DISTResult types in `types.py`, and unit tests with mocked dist-s1.
</objective>

Functions to implement:

- `validate_dist_product(cog_paths: list[Path]) -> list[str]` -- per D-13, reuse the same validation logic as `validate_rtc_product` in rtc.py (lazy import rasterio + rio_cogeo). Check each file: exists, valid COG (cog_validate), UTM CRS (EPSG 326xx or 327xx), pixel size 25-35m. Return list of error strings (empty = valid). This is a separate function (not imported from rtc.py) to keep modules independent.

- `run_dist(mgrs_tile_id: str, post_date: str, track_number: int, output_dir: Path) -> DISTResult` -- per D-11, the low-level entry point:
  1. `output_dir.mkdir(parents=True, exist_ok=True)`
  2. Try lazy import: `from dist_s1 import run_dist_s1_workflow`. On ImportError, raise `ImportError("dist-s1 is not installed. Install via conda-forge: mamba install -c conda-forge dist-s1")`.
  3. Call `run_dist_s1_workflow(mgrs_tile_id=mgrs_tile_id, post_date=post_date, track_number=track_number, dst_dir=output_dir)`.
  4. Collect output COG GeoTIFFs: `sorted(output_dir.glob("*.tif"))`.
  5. Validate: `errors = validate_dist_product(cog_paths)`.
  6. Return `DISTResult(output_paths=cog_paths, output_dir=output_dir, valid=len(errors)==0, validation_errors=errors)`.
  7. Wrap in try/except for the dist_s1 call itself (not the import). On exception, log error and return invalid result with error message.

- `_aoi_to_mgrs_tiles(aoi: dict) -> list[dict]` -- per D-12, resolve an AOI GeoJSON geometry to a list of MGRS tile IDs and their associated track numbers. Use `from pyproj import CRS` and basic UTM zone math from the AOI bounds to determine the relevant MGRS grid zone designator(s). Return a list of dicts: `[{"mgrs_tile_id": "33UUP", "track_number": 95}, ...]`. For the initial implementation, use a simplified mapping based on the AOI centroid longitude to UTM zone, then construct MGRS tile IDs from the UTM zone + latitude band. This is a best-effort mapping -- the actual dist-s1 workflow will validate tile availability.

- `run_dist_from_aoi(aoi: dict | Path, date_range: tuple[str, str], output_dir: Path) -> list[DISTResult]` -- per D-12, the end-to-end AOI+date range entry point:
  1. `aoi` is either a GeoJSON dict (with "type": "Polygon" and "coordinates") or a Path to a GeoJSON file. If Path, load with `json.load()`. Validate it has "type" and "coordinates" keys.
  2. Resolve MGRS tiles for the AOI: call `_aoi_to_mgrs_tiles(aoi)`.
  3. Build RTC time series first: Use Phase 1/2 data layer to run RTC for the AOI:
     a. Query burst DB: `from subsideo.burst.db import query_bursts` to get burst IDs.
     b. Use CDSEClient to search for S1 IW SLC scenes within `date_range`: `from subsideo.data.cdse import CDSEClient`.
     c. Fetch DEM: `from subsideo.data.dem import fetch_dem`.
     d. For each scene, fetch orbit: `from subsideo.data.orbits import fetch_orbit`.
     e. For each scene/burst, call `from subsideo.products.rtc import run_rtc`. Log progress.
  4. For each MGRS tile + date in the range, call `run_dist(mgrs_tile_id=tile["mgrs_tile_id"], post_date=date_range[1], track_number=tile["track_number"], output_dir=output_dir / tile["mgrs_tile_id"])`.
  5. Collect and return all DISTResult objects.
  Log progress at each stage: "Resolved {n} MGRS tiles for AOI", "Building RTC time series: {i}/{n} scenes...", "Running DIST-S1 for tile {tile_id}...".

Use `from __future__ import annotations`. Use loguru logger. All dist-s1 and rio_cogeo imports must be lazy (inside function bodies). Phase 1/2 imports inside run_dist_from_aoi should also be lazy to keep the module importable.
