<objective>
Build the EU-scoped Sentinel-1 burst SQLite database and the query layer that resolves AOI geometries to burst IDs.

Purpose: BURST-01/02/03 are the critical data-routing layer for all Phase 2+ pipeline work. Every RTC and CSLC pipeline invocation needs burst IDs and UTM zone assignments from this database. The key constraint: opera-burstdb covers North America only — we must build our own EU-scoped version from ESA's GeoJSON using the same schema.
Output: `BurstDB` build pipeline, `query_bursts_for_aoi()` spatial resolver, `select_utm_epsg()` zone selector, and unit tests using in-memory SQLite (no network required). Note: D-05 requires "CLI or programmatic" rebuild entry point — `build_burst_db()` is the programmatic entry point and fully satisfies D-05. A CLI wrapper can be added in a later phase if desired.

**src/subsideo/utils/projections.py:**

Implement `utm_epsg_from_lon(lon: float, lat: float = 45.0) -> int` using pyproj. Do NOT use the naive `int((lon + 180) / 6) + 1` formula — it fails for Norway/Svalbard UTM anomalies. Instead:
```python
from pyproj import CRS
from pyproj.aoi import AreaOfInterest
from pyproj.database import query_utm_crs_info

def utm_epsg_from_lon(lon: float, lat: float = 45.0) -> int:
    """Return UTM EPSG code using pyproj (handles zone anomalies for Norway/Svalbard)."""
    results = query_utm_crs_info(
        datum_name="WGS 84",
        area_of_interest=AreaOfInterest(
            west_lon_degree=lon,
            south_lat_degree=lat - 0.5,
            east_lon_degree=lon,
            north_lat_degree=lat + 0.5,
        ),
    )
    if not results:
        raise ValueError(f"No UTM CRS found for lon={lon}, lat={lat}")
    return int(results[0].code)
```

Implement `build_burst_db(geojson_source, output_path=None, eu_bounds=(-32.0, 27.0, 45.0, 72.0)) -> Path`:

1. Load GeoJSON: `gdf = gpd.read_file(geojson_source)` (geopandas handles both URLs and local paths)
2. Filter to EU bounds using a bounding box: `from shapely.geometry import box; eu_box = box(*eu_bounds); gdf = gdf[gdf.geometry.intersects(eu_box)]`
3. Ensure CRS is EPSG:4326: `if gdf.crs != "EPSG:4326": gdf = gdf.to_crs("EPSG:4326")`
4. Build records: For each row in gdf, construct a `BurstRecord`. The EPSG must come from `utm_epsg_from_lon(centroid.x, centroid.y)` where centroid is the burst polygon centroid. Store the EPSG in the DB — never derive it at query time. Set `is_north=1` for all EU bursts (all are in northern hemisphere zones).
5. Map GeoJSON fields to schema: GeoJSON uses ESA burst ID conventions. The `burst_id_jpl` should be constructed as `f"T{relative_orbit:03d}-{burst_index:06d}-{subswath}"`. Inspect actual GeoJSON field names (common: `relative_orbit_number`, `burst_index`, `subswath_name` or `IW`) when implementing — adapt to actual field names found.
6. Write to SQLite: Create the output directory, open sqlite3 connection, create table:
```sql
CREATE TABLE IF NOT EXISTS burst_id_map (
    burst_id_jpl TEXT PRIMARY KEY,
    burst_id_esa TEXT NOT NULL,
    relative_orbit_number INTEGER NOT NULL,
    burst_index INTEGER NOT NULL,
    subswath TEXT NOT NULL,
    geometry_wkt TEXT NOT NULL,
    epsg INTEGER NOT NULL,
    is_north INTEGER DEFAULT 1
);
CREATE INDEX IF NOT EXISTS burst_spatial ON burst_id_map(epsg);
```
Insert all records. Add `-- ESA CC-BY 4.0` comment in the build function docstring.
7. Set `output_path = output_path or get_burst_db_path()`. Create parent dirs. Return the output path.

```python
"""UTM zone selection for EU bursts."""
from __future__ import annotations
from subsideo.burst.db import BurstRecord


def select_utm_epsg(burst_record: BurstRecord) -> int:
    """Return the UTM EPSG code for a burst.

    Reads directly from the burst record (pre-stored at DB build time).
    NEVER derives UTM zone from coordinates at query time — EU spans
    zones 28N–38N (EPSG 32628–32638) and coordinate-based derivation
    fails for Norway/Svalbard anomalies.
    """
    return burst_record.epsg
```
