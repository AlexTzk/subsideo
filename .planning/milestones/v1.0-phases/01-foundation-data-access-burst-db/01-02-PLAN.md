Implement the CDSE data access client for Sentinel-1 and Sentinel-2 searches and downloads.

Purpose: DATA-01 and DATA-02 require searching and downloading from the Copernicus Data Space Ecosystem. The CDSE S3 endpoint is non-standard (the #1 integration gotcha from research). Centralising all CDSE S3 configuration in one class prevents the most common failure mode: scattered boto3 calls without the custom endpoint.

**Implementation (src/subsideo/data/cdse.py):**

Module-level constants:
```python
CDSE_TOKEN_URL = "https://identity.dataspace.copernicus.eu/auth/realms/CDSE/protocol/openid-connect/token"
CDSE_STAC_URL = "https://stac.dataspace.copernicus.eu/v1"
CDSE_S3_ENDPOINT = "https://eodata.dataspace.copernicus.eu"
```

`CDSEClient.__init__`: Store `_client_id` and `_client_secret`. Do NOT fetch OAuth2 token at init (lazy auth).

`CDSEClient._get_token()` (private method): Fetch OAuth2 token using `requests_oauthlib` per D-02:
```python
from requests_oauthlib import OAuth2Session
from oauthlib.oauth2 import BackendApplicationClient

def _get_token(self) -> str:
    client = BackendApplicationClient(client_id=self._client_id)
    oauth = OAuth2Session(client=client)
    token = oauth.fetch_token(
        token_url=CDSE_TOKEN_URL,
        client_id=self._client_id,
        client_secret=self._client_secret,
    )
    return token["access_token"]
```

`CDSEClient.s3_client()` (private method): Returns `boto3.client("s3", endpoint_url=CDSE_S3_ENDPOINT, region_name="default", aws_access_key_id=self._client_id, aws_secret_access_key=self._client_secret)`. This is the ONLY place `boto3.client("s3")` is called. S3 auth uses client_id/secret directly (not bearer token) — separate auth mechanism.

`CDSEClient.search_stac()`: Uses `pystac_client.Client.open(CDSE_STAC_URL)`. Call `.search(collections=[collection], bbox=bbox, datetime=f"{start.isoformat()}Z/{end.isoformat()}Z", query={"productType": {"eq": product_type}} if product_type else {})`. If `max_items` is set, pass it. Return `list(results.items_as_dicts())`. Works identically for `collection="SENTINEL-1"` and `collection="SENTINEL-2"` — the collection parameter is passed through without special-casing.

`CDSEClient.download()`: Parses `s3_path` by stripping `"s3://"` and splitting on first `/` to get bucket (`"eodata"`) and key. Creates `output_path.parent.mkdir(parents=True, exist_ok=True)`. Implements retry loop: `for attempt in range(max_retries): try: self.s3_client().download_file(bucket, key, str(output_path)); return output_path; except ClientError as e: wait = min(2**attempt, 60); time.sleep(wait)`. After loop exhausted, raises `RuntimeError(f"CDSE S3 download failed after {max_retries} retries: {s3_path}")`.

`CDSEClient.verify_connectivity()`: Checks `self._client_id` and `self._client_secret` non-empty, raises `ValueError` with remediation message if not. Then attempts a single STAC catalog `Client.open(CDSE_STAC_URL)` to verify network reachability.

Imports needed: `from __future__ import annotations`, `import time`, `from datetime import datetime`, `from pathlib import Path`, `import boto3`, `from botocore.exceptions import ClientError`, `from pystac_client import Client`, `from requests_oauthlib import OAuth2Session`, `from oauthlib.oauth2 import BackendApplicationClient`, `from loguru import logger`
