**B-01 (D-01, D-02): CDSEClient credentials.** Replace:
```python
client = CDSEClient()
```
With:
```python
from subsideo.config import Settings
settings = Settings()
client = CDSEClient(
    client_id=settings.cdse_client_id,
    client_secret=settings.cdse_client_secret,
)
```
Add the `Settings` import as a lazy import inside the function body (matching established pattern). In rtc.py this is around line 310; in cslc.py around line 282.

**B-02 (D-03): search_stac method and args.** Replace:
```python
items = client.search(
    collection="sentinel-1-slc",
    bbox=geom.bounds,
    datetime_range=(start_dt, end_dt),
)
```
With:
```python
items = client.search_stac(
    collection="SENTINEL-1",
    bbox=list(geom.bounds),
    start=start_dt,
    end=end_dt,
    product_type="IW_SLC__1S",
)
```
Note: `geom.bounds` returns a tuple; wrap in `list()` to match the `list[float]` type hint.

**B-03 (D-04): Burst DB import.** Replace the block:
```python

burst_ids = db.query_by_geometry(geom.wkt)
```

With:
```python

burst_ids = [b.burst_id_jpl for b in bursts]
```
Also add an early-exit guard for empty bursts (Pitfall 3 from RESEARCH.md):

```python
if not bursts:
```

**B-04 (D-05): fetch_orbit args.** Replace:
```python
orbit_path = fetch_orbit(safe_paths[0])
```
With (extract metadata from STAC item, using `items[0]` since RTC/CSLC process single scene per Pitfall 5):
```python
scene = items[0]
sensing_time = datetime.fromisoformat(
    scene.get("properties", {}).get("datetime", date_range[0]).rstrip("Z")
)
satellite = scene.get("properties", {}).get("platform", "S1A")
orbit_path = fetch_orbit(
    sensing_time=sensing_time,
    satellite=satellite,
    output_dir=output_dir / "orbits",
)
```
The `.rstrip("Z")` handles Python 3.10 compatibility (Pitfall 4 / Open Question 1 from RESEARCH.md).

**B-05 (D-06, D-07): fetch_dem with output_epsg and tuple unpack.** Replace:
```python
dem_path = fetch_dem(geom.bounds, output_dir / "dem")
```
With:
```python
output_epsg = bursts[0].epsg
dem_path, _dem_profile = fetch_dem(
    bounds=list(geom.bounds),
    output_epsg=output_epsg,
    output_dir=output_dir / "dem",
)
```
This must come AFTER the B-03 fix (which provides `bursts`). Reorder the data-access block so burst query comes before DEM fetch.

```python
s3_key = scene.get("assets", {}).get("data", {}).get("href", "")
```

**Reorder** the data-access block in both files to: (1) Settings + CDSEClient, (2) search_stac, (3) burst query, (4) DEM fetch, (5) orbit fetch, (6) download, (7) run algorithm. This ensures burst records are available before DEM fetch needs `.epsg`.

Keep the single-scene behavior (process `items[0]` only) per Pitfall 5 -- this phase is contract alignment, not feature addition.
